# Contributing to Doc Repo

## Principles

- **Teach, don’t market.** Prefer accuratePlain explanations over hype.
- **Link to code** for volatile details (exact routes, env names)—duplicating them here guarantees drift.
- **No secrets** — ever. Use placeholders like `<SET_IN_CLOUDFLARE>`.

## Style

- Short sections with tables where they clarify comparisons.
- One idea per H2; avoid giant walls of text.

## Pull requests

1. Fork or branch from `main`.
2. Update **`README.md`** ToC if you add top-level sections.
3. Run spell-check—many readers are non-native English speakers.

## Fact-checking

If you change architecture claims, cite the file path in the web/mobile repo in your PR description so reviewers can verify quickly.
